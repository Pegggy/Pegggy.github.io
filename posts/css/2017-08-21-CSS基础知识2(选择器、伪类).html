1.class 和 id 的使用场景?

class是类选择器，用于页面中较多元素需采用同一种样式，可以为任意多个元素指定相同的class，也可以为同一个元素指定多个class，只需要用空格分隔即可。

id选择器是定向的为页面中唯一的某个元素指定样式，id选择器不可结合使用，不允许出现以空格分隔的词列表。

复杂场景优先级计算

  • 行内样式
    ==> 1，0，0，0
  • ID 选择器 ==> 0，1，0，0
  • 类，属性选择器和伪类选择器 ==> 0，0，1，0

4.a:link, a:hover, a:active, a:visited 的顺序

a:linka标签初始样式，放于最上部；
a:visited a标签访问后的状态，位于初始样式之后；
a:active鼠标点击时的状态，位于样式最底部；
a:hover鼠标移动过a标签的状态，位于a:visited之后。

因为后面的样式会覆盖前面的样式，a标签初始状态样式应该放于最顶部，其他状态都可以覆盖初始状态样式。

a:visited需放置在初始样式之下以避免对hover和active状态产生影响。若置于最后，则一次点击之后，无论再做任何操作，a标签的样式一直都会处于:visited的样式,因为此时将:active、:hover状态都覆盖了。

同理，:active状态需放置在:hover状态后面。
